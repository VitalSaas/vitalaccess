# Contributing to VitalAccess

Thank you for considering contributing to VitalAccess! We welcome contributions from the community.

## 📋 Code of Conduct

Please be respectful and constructive in all interactions.

## 🐛 Bug Reports

When filing a bug report, please include:

- A clear description of the issue
- Steps to reproduce the problem
- Expected vs actual behavior
- Your environment details (PHP version, Laravel version, etc.)
- Any relevant error messages or logs

## 🚀 Feature Requests

We welcome feature requests! Please:

- Search existing issues first
- Describe the feature and its use case
- Explain why it would be valuable
- Consider submitting a PR if you can implement it

## 🔧 Development Setup

1. Clone the repository
2. Install dependencies: `composer install`
3. Run tests: `composer test`
4. Follow PSR-12 coding standards

## 📝 Pull Requests

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Add tests for new functionality
5. Ensure all tests pass
6. Update documentation if needed
7. Submit a pull request

## 🧪 Testing

- Run tests with `composer test`
- Add tests for new features
- Ensure all existing tests still pass

## 📖 Documentation

- Update README.md for new features
- Add docblocks to new methods
- Update CHANGELOG.md

## ✅ Code Style

- Follow PSR-12 standards
- Use meaningful variable and method names
- Keep methods focused and small
- Add comments for complex logic

Thank you for contributing! 🎉
